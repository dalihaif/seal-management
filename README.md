# 🔖 印章管理系统

大理大学第一附属医院 — 印章全生命周期轻量化管理系统

> 纯前端 + Flask 后端，单文件部署，支持离线使用，可连接医院 OA 系统

---

## ✨ 功能特性

### 核心功能
| 模块 | 功能 |
|------|------|
| 📋 印章台账 | 新增/编辑/删除印章，支持批量操作 |
| 🗄️ 批量归档 | 多选印章统一归档，生成归档证明（打印/导出CSV） |
| 🖊️ 使用登记 | 扫码快速登记用印，支持「借出/归还/盖章使用」三种类型 |
| 📊 报表导出 | 导出 Excel 报表，支持按类型/部门/状态筛选 |
| 🔍 高级搜索 | 按名称/编号/类型/部门/管理人多维搜索 |
| 📁 数据导入 | 支持 .xlsx / .xls 批量导入印章数据 |

### v3.4 新增
- ✅ **批量归档重构**：独立弹窗，支持多选印章统一归档
- ✅ **归档证明修复**：修复打印空白问题
- ✅ **归档导出**：支持打印证明 + CSV 导出双模式
- ✅ **印章类型统一**：16 种类型全局常量驱动，新增类型只需改一处

## 🚀 快速启动

### 环境要求
- Python 3.8+
- Flask (`pip install flask`)

### 启动步骤

```bash
# 1. 克隆仓库
git clone https://github.com/dalihaif/seal-management.git
cd seal-management

# 2. 安装依赖
pip install flask

# 3. 启动服务（默认端口 5100）
python server.py

# 4. 浏览器访问
# http://localhost:5100
```

### 一键启动（Windows）

双击 `start.bat` 即可自动启动服务并打开浏览器。

## 📁 项目结构

```
seal-management/
├── index.html          # 主页面（管理端，单文件）
├── usage.html          # 使用登记端（扫码访问）
├── server.py           # Flask 后端服务
├── data.json           # 印章数据（自动同步至 SQLite）
├── seal_archive.db     # SQLite 数据库
├── export_data.py      # 数据导入脚本
├── enrich_data.py      # 数据丰富化脚本
├── start.bat           # Windows 一键启动
└── 印章目录.xlsx       # 原始Excel数据
```

## 🔧 印章类型（全局定义）

在 `index.html` 中通过 `SEAL_TYPES` 常量统一管理，目前支持 **16 种类型**：

```
公章、法人章、财务章、合同章、部门章、其他、
住院结算专用章、收费专用章、财务专用章、
出生医学证明专用章、发票专用章、机要文件收发章、
病历复印专用章、医疗证明章、经济合同专用章、体检专用章
```

> 💡 新增类型只需修改 `index.html` 中的 `SEAL_TYPES` 数组，全系统自动同步。

## 📊 数据字段

| 字段 | 说明 |
|------|------|
| id | 编号（自动分配） |
| name | 印章名称 |
| type | 印章类型（16种） |
| dept | 保管部门 |
| keeper | 管理人 |
| material | 材质（橡胶/木质/金属/牛角/自动印章） |
| shape | 形状（圆形/椭圆/方形/长方形） |
| carvingDate | 刻制日期 |
| startDate | 启用日期 |
| endDate | 废止日期 |
| validUntil | 有效期至 |
| location | 存放位置 |
| status | 状态（在用/停用/已废止/已销毁/已归档） |
| remark | 备注 |
