# 印章管理系统

**大理大学第一附属医院印章管理系统** — 基于 Flask + SQLite 的印章全生命周期管理平台。

## 功能模块

| 模块 | 说明 |
|------|------|
| 印章目录 | 印章台账管理，支持新增/编辑/停用/启用，印章分类筛选 |
| 刻制管理 | 印章刻制申请、审批流程、刻制进度跟踪 |
| 使用登记 | 印章使用记录，支持多印章同时使用、使用事由/用途登记 |
| 借用管理 | 印章借出外带管理，借还流程、到期提醒、一键归还 |
| 回收核验 | 印章回收检查与核验记录 |
| 作废销毁 | 印章作废与销毁审批、执行记录 |
| 归档记录 | 印章归档管理，支持批量归档、归档证明导出 |
| 标签打印 | 印章标签与档案标签打印 |
| 统计报表 | 各维度数据统计与可视化 |
| 系统设置 | 部门管理、审批人配置、默认印章设置、用户管理 |
| 操作日志 | 全量操作记录追溯 |

## 用户与权限

系统内置三种角色，支持密码登录和免密进入：

| 角色 | 用户名 | 默认密码 | 权限 |
|------|--------|----------|------|
| 经办人 | clerk | 免密进入 | 查看印章、使用登记、借用管理、标签打印、统计报表 |
| 印章管理员 | editor | edit123 | 全部管理功能（含印章增删改、审批、导入导出） |
| 系统管理员 | admin | admin123 | 全部权限 + 用户管理 + 系统设置 |

- 首次使用账号密码登录时会提示修改默认密码
- 经办人可直接点击「以经办人身份进入」免密登录
- 所有用户均可在系统中修改自己的密码

## 技术栈

- **后端**：Python 3.8+ / Flask
- **数据库**：SQLite
- **前端**：原生 HTML/CSS/JavaScript（单页应用）
- **部署**：一键启动脚本 `start.bat`

## 快速开始

### 环境要求

- Python 3.8 及以上
- pip（Python 包管理器）

### 安装与启动

```bash
# 1. 克隆仓库
git clone https://github.com/dalihaif/seal-management.git
cd seal-management

# 2. 安装依赖
pip install flask

# 3. 启动系统
python server.py
```

或直接双击 `start.bat` 自动完成环境检查、依赖安装并启动服务。

启动后浏览器访问：**http://localhost:5100**

首次启动会自动初始化数据库并导入演示数据（10条印章 + 各模块演示记录）。

## 主要特性

- **全流程管理**：覆盖印章从刻制到销毁的完整生命周期
- **用户认证**：基于角色的登录系统，支持密码登录和免密经办人模式
- **权限控制**：三种角色（经办人/印章管理员/系统管理员），精细化功能权限
- **审批流程**：多级审批人配置，支持院长/副院长/科室主任分级审批
- **多印章使用**：支持一次登记多个印章同时使用
- **借用外带**：借出审批、到期跟踪、一键归还
- **批量操作**：批量选择、批量修改状态、批量删除、批量导出 CSV
- **条数统计**：各模块实时显示当前筛选条件下的记录条数
- **CSV 导出**：所有模块支持按当前筛选条件导出报表
- **标签打印**：印章标签和档案标签打印功能
- **操作日志**：完整记录所有操作，支持追溯审计
- **数据看板**：首页实时统计印章总数、使用中、借出中、待审批等状态

## 项目结构

```
├── export_data.py      # 数据导出工具

```
